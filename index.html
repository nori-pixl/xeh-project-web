統合マルチランタイム・オーケストレーション言語

xeh は、XMLベースの構成と柔軟なデータストアを基盤に、複数の異なる言語ランタイムを同一メモリ空間で安全に同時実行・制御するためのメタ言語エンジンです。

xeh download

ダウンロードとビルド

xeh は公式リポジトリからソースコードを取得し、Go環境を使って簡単にビルドすることができます。お使いのOSに合わせてコマンドを実行してください。

Linux / macOS

bash# リポジトリをクローン
git clone https://github.com/nori-pixl/xeh-project
cd xeh-project

# バイナリをビルド
go build -o xeh main.go

# パスの通った場所に配置（Linux/Mac共通）
sudo mv xeh /usr/local/bin/

Windows (PowerShell)

powershell# リポジトリをクローン
git clone https://github.com/nori-pixl/xeh-project
cd xeh-project

# 実行ファイル(.exe)をビルド
go build -o xeh.exe main.go

# ユーザー環境変数のパスが通ったフォルダ（例: C:\Windows）等に配置
# ※または環境変数 PATH に xeh.exe のフォルダを追加してください
Move-Item xeh.exe C:\Windows\System32\ -Force

正常にインストールされたか確認します（全OS共通）：

bash$ xeh --version
xeh version 1.0.0

仕組みと使い方（クイックスタート）

xehアプリケーションは、ランタイムやマッピング環境を定義する set.json と、データ・制御ロジックを統合管理（オーケストレーション）する app.xeh の一対の構成で駆動します。

2. 処理ロジックとメモリの記述 (app.xeh)

共通で使用する変数をJSON形式で割り当て（アロケート）、対応するサブランタイムのコードをXMLタグ内に内包させます。

xml<xeh>
  <root memory_name="production_space">
    [
      {"name": "threshold", "type": "int", "value": 85},
      {"name": "status", "type": "string", "value": "active"}
    ]
  </root>

  <subframework roteapp="main">
    <!-- Pythonインタプリタに内包コードを転送 -->
    <py-processor>
import sys, json
# 標準入力(STDIN)から共通メモリ(MemoryPacket)のJSONデータを取得
packet = json.loads(sys.stdin.read())
print(f"[Python Subprocess] 共有しきい値: {packet['data']['threshold']}")
    </py-processor>

    <!-- xeh標準のネイティブXMLロジック -->
    <xeh-logic>
      <print value="現在のステータス: \${status}" />
    </xeh-logic>
  </subframework>
</xeh>

アーキテクチャと機能の特徴

Go言語の強力な並行処理と標準ライブラリの恩恵をフルに活かした設計になっています。

並行プロセス制御

Goの並行処理（sync.WaitGroup）を活用。複数の外部ランタイムプロセスを非同期に立ち上げ、ノンブロッキングで同時に並行動作させます。

スレッド安全な共有メモリ

内蔵データストアは sync.RWMutex で厳重に保護。各サブプロセスへ安全に MemoryPacket（JSON）として最新状態が引き渡されます。

クリーンな強制終了

os/signal 監視機構により、Ctrl+C等による中断時も、実行中の子プロセスを漏れなく確実に強制終了（Kill）し、ゾンビプロセスの発生を防ぎます。

プラグイン機構

<subframework import="..."> 機能を搭載。外部の .xeh 資産から動的に変数やモジュールをロードして統合できます。

技術ドキュメント

xehの言語仕様、およびエンジン実装の詳細仕様に関するリファレンスです。

  • メモリ＆標準入力（STDIN）アーキテクチャ詳細仕様 - インタプリタ間データ受け渡しのシリアライズ規則
  • 組み込みXMLタグリファレンス (<xeh-logic>) - ネイティブタグと変数展開マクロ（\${var}）の仕様
  • プラグインと共有ストアのカスタマイズ - 外部パッケージのインポートと変数マージ規則
